Project Overview

This was a team project that involved all members of the design team.

My Role

I was one of the UX/UI designers, I was responsible for mockup and prototype creation aspects of this project, with a focus on image layer UI.

Tools

Figma, Maze, Figjam

Challenge

Viewing imagery is is a core piece of the business model, but the experience was disjointed, confusing, and hard to navigate.

In addition, there was no way to compare imagery. A user could only view one surface layer at a time and toggle between.

Approach

We needed to first understand the scope of the problem and then ask customers about their pain points and desires.

As a team, we mapped out the entire end to end experience. Then interviewed power users to define key principles and moments of the new experience.

Results

FieldAgent will soon release a new field UI with new imagery handling capabilities. In the UI a user will be able to not only see a comprehensive field overview, but view and compare imagery layers.

This was a customer pain point for a long time and a huge win for the business.

Analy t

We researched our competitors to identify trends and market gaps.

Additionally, we interviewed users who regularly used the FieldAgent interface. The interviews walked through the current flow and gave interviewees opportunity to talk through pain points and offer insights.

We quickly saw trends begin to crystallize and began planning for the design sprint. A design sprint would allow us to collaborate and arrive at a concept for user testing.

Testing the Concept

Recruiting users to test has been a huge challenge for the team. We simply do not have a massive user base to recruit from, so we went a different route. We first recruited users to a product council that set the table stakes for participation and allowed us to start a database of willing testers.

Maze is a user testing platform that allows us to send unproctored tests via email, so that the testing experience was convenient and fast.
